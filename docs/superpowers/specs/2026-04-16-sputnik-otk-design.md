# Sputnik OTK — спецификация приложения

Дата: 2026-04-16
Автор: Татьяна Кондушова + Claude
Статус: черновик, на ревью

## 1. Цель

Заменить функциональный блок «ОТК» из старого Android-приложения (`com.example.my_application`, APK без исходников) на новое самостоятельное Android-приложение, которое записывает результаты контроля качества в Google Sheets через Apps Script webhook.

Старый APK продолжает работать и используется для блока SK-Warranty (Bitrix). Новое приложение устанавливается параллельно, ничего в старом не ломает.

## 2. Контекст

- Старый APK содержит два блока: SK-WARRANTY (Bitrix) и ОТК (Google Sheets). Блок ОТК перестал работать после отзыва ключа сервисного аккаунта Google.
- Google Cloud Project для сотрудника более не доступен (нет админского доступа и страна пользователя закрыта для регистрации новых GCP-проектов).
- Решение: заменить сервисный аккаунт на Google Apps Script webhook, живущий внутри Google-таблицы.
- Webhook реализован и проверен (скрипт `webhook.gs` в Apps Script проекте `SK-Warranty-Sheets`), принимает JSON с полем `password` и опциональными полями `panel`, `master`, `date`, `fault`, `note`, `tested`.
- Задача «переписать приложение целиком» поставлена руководством как учебная для каждого сотрудника без найма фрилансеров. Пользователь — новичок в программировании.

## 3. Scope

### В этой спеке (первый этап)

Только блок **ОТК**: самостоятельное Android-приложение, пишущее строки в Google Sheets через webhook.

Главный экран приложения с выбором блоков задумывается так, чтобы в будущем к нему можно было добавить блок SK-Warranty без переделки навигации. Сейчас на главном экране — только плитка «ОТК».

### НЕ в этой спеке (второй этап, отдельная спека)

- Блок SK-Warranty (интеграция с Bitrix).
- Авторизация (логин/пароль на входе в приложение).
- Аналитика, отчёты, административные экраны.
- Кросс-платформенность (iOS).

## 4. Пользователи и рабочий процесс

### Пользователи

- **4 штатных контроллёра ОТК**: Руслан, Камиль, Виктор, Тимур.
- **«Мастер»** — пятый пункт выпадающего списка для подменяющего человека (например, руководителя, который вне графика занялся проверкой). Не привязан к конкретному сотруднику.
- **Выбор мастера обязателен** перед добавлением панелей. При запуске приложения ничего не выбрано — ввод номера, кнопка ➕ и приём NFC заблокированы. Выбор действует только до закрытия приложения и **не сохраняется** между запусками (защита от случайной подписи чужим именем, особенно после сессии под «Мастером»).
- **Татьяна** — владелец таблицы, вносит колонки E–F (замечания, что тестировали) вручную постфактум. Через скрытый экран настроек может менять URL webhook и пароль.

### Рабочий сценарий (смена контроллёра ОТК)

1. Контроллёр приходит на рабочее место со стендом на 8 панелей.
2. Устанавливает панели на стенд, выполняет проверку (прошивка камеры, домофон, звук, открывание двери, либо частичный набор тестов).
3. После завершения проверки **по очереди сканирует NFC-метку каждой панели**. Номер автоматически попадает в список на экране.
4. Если NFC не считался — вводит номер руками в постоянно видимое поле ввода.
5. Если есть замечание по конкретной панели — нажимает ✏, открывает диалог, вписывает комментарий в поле «Неисправность».
6. По готовности нажимает **«Сохранить»** — все записи из списка пачкой отправляются в таблицу через webhook.
7. Успешные удаляются из списка; неуспешные остаются с отметкой, плюс запись в «Логи».
8. Контроллёр берёт следующий стенд из 8 панелей, повторяет цикл.

За смену — **до 200 панелей** = 25 партий по 8 штук.

## 5. Требования к платформе

- **Android** 8.0 и выше (API 26+). Покрывает Huawei P30 Lite (Android 9/10) и все новые трубки сотрудников.
- **NFC** обязателен (есть у всех сотрудников).
- **Wi-Fi** всегда доступен на рабочих местах — оффлайн-режим и очередь отправки НЕ требуются.
- **Только Android**, iOS не в scope.

```
┌──────────────────────────────────┐
│  ОТК                🧹   📋      │  шапка + метёлка + копирование
├──────────────────────────────────┤
│                                  │
│  Мастер:  [ ▼ — Выбери —    ]    │  выпадашка, 5 вариантов, обязательно
│                                  │
│  Дата:    16.04.2026             │  авто, не редактируется
│                                  │
│  Номер панели:                   │
│  ┌──────────────────────────┐    │
│  │ _____________________  ➕│    │  ручной ввод + кнопка добавить
│  └──────────────────────────┘    │
│                                  │
│  💡 Поднеси телефон к метке —    │
│     номер добавится сам          │
│                                  │
│  Отсканировано (3):              │
│  ┌────────────────────────────┐  │
│  │ 04:A3:B2:F8:12:3D      ✏  │  │
│  │ 04:C1:D5:E2:91:AB  💬  ✏  │  │  💬 = есть коммент
│  │ 04:FE:12:34:56:78      ✏  │  │
│  └────────────────────────────┘  │
│                                  │
├──────────────────────────────────┤
│  [Сохранить]   [Логи]   [Назад]  │
└──────────────────────────────────┘
```

**Элементы:**

- **🧹 метёлка** — очищает список после подтверждающего диалога.
- **📋 копирование** — копирует все номера из списка в буфер обмена, по одному в столбик.
- **Выпадашка «Мастер»** — 5 вариантов: Руслан, Камиль, Виктор, Тимур, Мастер. При запуске **ничего не выбрано**, отображается placeholder «— Выбери мастера —». Пока мастер не выбран — поле «Номер панели», кнопка ➕ и приём NFC-интента заблокированы; попытка ввода показывает snackbar «Сначала выбери мастера». Выбор действует до закрытия приложения, после перезапуска сбрасывается.
- **Дата** — сегодняшняя, автоматически, только для отображения.
- **Поле «Номер панели»** — всегда видимо. Неактивно, пока мастер не выбран. После выбора — NFC заполняет его автоматически и тут же добавляет в список. Ручной ввод: набрать → нажать ➕ или Enter.
- **Защита от дубликатов:** при добавлении (NFC или ручной ввод) выполняется строковое сравнение по `Panel.id` с каждым элементом `pendingPanels`. При совпадении — вибрация и сообщение «Эта панель уже в списке», строка не добавляется.
- **Список** — каждая строка: номер, значок 💬 если есть коммент, ✏ для редактирования.
- **«Сохранить»** — неактивна, если список пуст или идёт отправка. При нажатии запускает последовательную отправку всех записей.
- **«Логи»** — открывает LogsScreen.
- **«Назад»** — возвращает на HomeScreen (плитки).

## 8. Данные и состояние

### Типы данных

```
Panel
  id: String           — номер панели (NFC UID или ручной ввод)
  fault: String        — комментарий (колонка D), "" если не задан
  addedAt: Long        — timestamp создания

ErrorEntry
  panelId: String
  reason: String
  timestamp: Long

Settings
  webhookUrl: String
  password: String

OtkUiState
  master: String?        — выбранный мастер, null пока не выбран
  pendingPanels: List<Panel>
  isSending: Boolean
  sendProgress: Pair<Int, Int>?   — "отправлено X из Y"
  snackbar: SnackbarData?
```

### Persistence

| Данные | Хранилище | Файл / ключ |
|--------|-----------|-------------|
| Список Panel (pending) | JSON-файл | `pending_panels.json` |
| Список ErrorEntry | JSON-файл | `error_log.json` |
| webhookUrl, password (override) | DataStore | `webhook_url`, `webhook_password` |

### Поток данных

- Compose-экран читает `OtkUiState` из `OtkViewModel` и отображает.
- Событие UI (тап, ввод) → ViewModel → обновляет state → репозиторий сохраняет в файл → экран автоматически перерисовывается.

## 9. Сеть и отправка

### Формат запроса

```
POST <webhookUrl>
Content-Type: application/json

{
  "password": "<password>",
  "panel":    "<panel.id>",
  "master":   "<selected master>",
  "date":     "YYYY-MM-DD (today)",
  "fault":    "<panel.fault or empty>"
}
```

Поля `note` и `tested` не посылаются — колонки E, F в таблице остаются пустыми, заполняются владельцем таблицы постфактум.

### Обработка ответа

| Ответ | Действие |
|-------|----------|
| HTTP 200 + `{"ok":true}` | Удалить `Panel` из `pendingPanels` |
| HTTP 200 + `{"ok":false,"error":"wrong password"}` | Прервать отправку остатка пачки, показать «Неверный пароль», запись в логи |
| HTTP 200 + `{"ok":false,"error":<other>}` | Оставить в списке, запись в логи с текстом ошибки |
| Network exception / timeout | Оставить в списке, запись в логи «нет связи с сервером» |

### Пачка

- Панели отправляются **последовательно** (одна за одной).
- Во время отправки кнопка «Сохранить» неактивна, отображается прогресс «X из Y».
- По завершении — snackbar: зелёный «Отправлено N из N ✓» либо красный «Отправлено X из Y, Z с ошибкой [Логи]».
- Отправка живёт в `viewModelScope`. Если пользователь явно закрывает приложение, корутина прерывается вместе с ViewModel — незавершённые записи остаются в `pendingPanels` и могут быть отправлены повторно после следующего запуска. Autosave после каждой успешной/неуспешной отдельной отправки гарантирует, что повторно уйдут только оставшиеся.

### Таймауты

- 30 секунд на запрос (OkHttp default).

### Google Apps Script особенность

Apps Script отвечает 302-редиректом на `script.googleusercontent.com/macros/echo?...`. OkHttp по умолчанию следует за редиректами и корректно переключается на GET при 302 — это поведение, которое даёт `{"ok":true}` (мы проверили на curl и Python).

## 10. Обработка ошибок

### Ожидаемые

| Ситуация | UX | Внутри |
|----------|-----|--------|
| Нет связи | Красный snackbar «Нет связи. Попробуй ещё раз.» | Запись в логи, строки остаются в списке |
| Неверный пароль | Красный snackbar с подсказкой открыть настройки | Отправка пачки прерывается |
| Ошибка сервера | Красный snackbar с текстом ошибки | Строка в логи, остаётся в списке |
| Дубликат панели | Вибрация + «Эта панель уже в списке» | Не добавляется в список |
| Мастер не выбран | Ввод/NFC заблокированы, snackbar «Сначала выбери мастера» | Событие добавления игнорируется |
| Пустой список | Кнопка «Сохранить» неактивна | — |
| NFC не считал | Ничего (стандартное поведение) | — |

### Неожиданные (краши)

- `Thread.setDefaultUncaughtExceptionHandler` ловит любое неперехваченное исключение.
- Полный stacktrace пишется в файл логов.
- Пользователю — «Что-то пошло не так. Перезапусти приложение. Если повторится — сообщи Татьяне.»
- State сохраняется до краша (autosave на каждое изменение).

## 13. Этапы реализации

Разбиваем на 8 шагов; каждый заканчивается устанавливаемым APK, на котором можно что-то потрогать.

1. **Пустой проект.** Android Studio создаёт каркас. Экран «Привет, ОТК». Проверяем цепочку: сборка → установка на телефон пользователя → запуск.
2. **HomeScreen.** Плитка «ОТК» + заголовок «Сервис ОТК» + переход на заглушку OtkScreen.
3. **OtkScreen базовый.** Выпадашка мастера (обязательный выбор, блокирует остальные поля до выбора; не сохраняется между запусками), ручной ввод номера, список, кнопки внизу. Кнопка «Сохранить» реально отправляет в webhook (пока без NFC, без карандашика, без логов). Первые реальные строки в таблице.
4. **NFC-сканирование.** Поддержка NDEF-интента, автоматическое добавление номера из метки.
5. **Карандашик + метёлка + копирование.** Редактирование комментариев, очистка списка, копирование в буфер.
6. **LogsScreen.** Персистентный список ошибок, кнопка очистки.
7. **SettingsScreen.** Скрытый экран с URL/паролем.
8. **Полировка.** Иконка, название, релизная подпись APK, финальная ручная проверка по чеклисту.

## 15. Принятые решения и trade-offs

| Решение | Почему |
|---------|--------|
| Kotlin + Jetpack Compose | Официальный современный стек, короче и понятнее XML для новичка. |
| Только Android | iOS-устройств у компании нет, cross-platform избыточен. |
| Отдельное приложение, а не замена старого | Требование «не ломаем рабочее». |
| Webhook вместо GCP service account | GCP недоступен пользователю; webhook не требует админа и пересоздаётся внутри таблицы. |
| Последовательная отправка пачки | Apps Script не любит параллельные запросы, для 8 штук разница незаметна. |
| Храним состояние в JSON-файлах, не Room | Room — дополнительная сложность, для объёма в 200 записей в день избыточна. |
| Хардкод конфига + скрытый экран настроек | Обычный сотрудник ничего не сломает; Татьяна может поменять без пересборки. |
| Оффлайн / очередь НЕ реализуем | Интернет всегда стабилен, YAGNI. |
| Список мастеров — хардкод в коде | Штат из 4 человек + «Мастер», меняется редко; добавление/удаление сотрудника = пересборка APK, это учебный проект, приемлемо. |
| Выбор мастера обязателен и не сохраняется между запусками | Защита от подписи чужим именем после сессии под «Мастером» (подмена). Лишний тап раз в день приемлем. |
| Пункт «Мастер» в списке имён | Универсальный слот для незапланированного контролёра (руководитель и т.п.), чтобы не заводить «временных» сотрудников в коде. |
